Catch the Tortoise Mahjong Game Site Plan

Aiden Bodily

WDD 131

Overview

Purpose

I want to make a fun browser game where you race against a bot to sort Mahjong tiles. You push blanks into columns to pop tiles out, flip them over, and try to get them all sorted before the bot does. Its supposed to be something quick and easy that anyone can just jump into without needing to know anything about Mahjong.

Audience

This is for anyone who likes puzzle games or sorting challenges. Mostly people looking for a quick game to play in their browser, like students on a break or anyone who enjoys strategy type games. You dont need to know Mahjong to play it.

Dynamic elements

I'm using JavaScript for pretty much the whole game. For DOM stuff I use getElementById to grab elements, then change their textContent and classList to update whats on screen, and addEventListener for all the click handling on the column buttons. Theres a lot of if/else branching because the game has different phases like when youre inserting a blank, placing a tile, or when the bot is taking its turn and each phase needs different logic. Each tile is stored as an object with a suit, number, and whether its face up or not, and I also have a tileEmojis object that maps each tile to its Mahjong emoji character. The grids are arrays of arrays (9 columns with 4 tiles each) and I use Array.from to create them, reduce and filter to count sorted tiles, and forEach to build the buttons and run the bot AI. Theres over 15 functions total for things like shuffling tiles, rendering the board, handling player clicks, bot AI, checking for winners, etc.

Navigation

Home

Theres only one page since its a single page game. The nav just links back to the game itself.

Content

Home page

The page has a header with the game title and a subtitle, and below that theres an about section with a hero image of a tortoise and a couple paragraphs explaining how the game works. The main game area has a turn indicator, move counter, and a new game button at the top, then the bot's grid, a status message area in the middle showing what tile you need to place, and then the player's grid below that. At the bottom theres a quick rules section so people can figure out how to play right away and a footer with a copyright and link to another game. For images I have an SVG hero illustration of a tortoise and I use Mahjong unicode emoji characters for the actual tiles in the game. The game data is 72 tile objects (9 numbers x 4 copies x 2 suits) stored in two 9-column grids.
